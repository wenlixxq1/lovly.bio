# 🚀 Настройка GitHub Pages для Next.js проекта

## 📝 Что нужно сделать:

### 1. Настроить GitHub Pages
1. Загрузите проект в ваш GitHub репозиторий
2. Зайдите в Settings вашего репозитория
3. Перейдите в раздел "Pages"
4. В "Source" выберите "GitHub Actions"

### 2. Файлы уже настроены:
- ✅ `next.config.js` - конфигурация для статического экспорта
- ✅ `.github/workflows/deploy.yml` - автоматическая сборка и деплой
- ✅ `.nojekyll` - отключает Jekyll обработку
- ✅ `utils/paths.ts` - правильные пути для ресурсов

### 3. Исправленные проблемы:
- ✅ Все изображения теперь используют правильные пути через `getImagePath()`
- ✅ Все видео используют правильные пути через `getVideoPath()`
- ✅ Убрано отсутствующее видео `cards-video.webm` из Skills компонента
- ✅ Настроена универсальная конфигурация для любого репозитория

### 4. После загрузки в репозиторий:
1. Сделайте push в ветку main
2. GitHub Actions автоматически соберет и задеплоит сайт
3. Сайт будет доступен по адресу: `https://ваш-username.github.io/имя-репозитория`

## 🔧 Проблема с изображениями решена!

Все ресурсы (изображения и видео) теперь будут корректно загружаться на GitHub Pages.

### Если изображения все еще не отображаются:
1. Убедитесь, что все файлы из папки `public/` загружены в репозиторий
2. Проверьте, что GitHub Actions успешно выполнился (вкладка Actions в репозитории)
3. Подождите несколько минут после деплоя для обновления кеша
